# 下一步执行计划

> 目标：在现有 Electron 桌面客户端基础上，继续把离线语音部署能力与产品体验推进到更接近“开箱可用”的状态。

## 当前基线

目前已完成：
- CosyVoice SFT 主链路打通
- TXT 分章增强
- loading / 进度反馈增强
- 长文本 TTS：首段优先、后台预取、磁盘缓存、动态预取窗口、manifest / 缓存治理
- 设置中心收拢
- 页面布局重构
- 书籍删除功能
- 模型管理 / 部署控制台
- 下载 / 安装任务化
- task templates + asset manifests
- 文件级校验框架与 asset 详情展示
- 任务持久化、重试机制、基础筛选
- CosyVoice 官方优先 + 手动导入模型目录 / 权重目录兜底
- `COSYVOICE_SFT_MODEL_DIR` 已进入启动路径与任务校验优先级

## 后续任务排序

### P1. CosyVoice 半自动安装体验收尾
目标：把当前“官方优先 + 手动导入兜底”的半自动安装方案做得更可感知、更顺手。

子项：
1. 页面更醒目显示“当前采用策略”
   - 官方源
   - 镜像源
   - 手动导入
2. 手动导入后立即校验
   - 路径存在性
   - 关键文件存在性
   - SFT 模型可用性
3. 手动导入后快速启动
   - 导入后提供一键验证 / 一键启动
4. 失败时更清晰提示
   - 缺目录
   - 缺权重
   - 缺脚本
   - 缺依赖

验收标准：
- 用户在页面中能明确看出当前使用哪条策略
- 手动导入后无需再猜测是否生效
- 导入 → 校验 → 启动链路在页面内可见

---

### P2. GPT-SoVITS 第二梯队接入
目标：在 CosyVoice 主路线稳定后，将 GPT-SoVITS 以第二梯队方式接入产品体验。

子项：
1. 统一模型控制台中的 GPT-SoVITS 策略说明
2. 明确参考音频 / 权重目录 / 启动入口
3. 保持“手动导入兜底”一致体验
4. 不抢主链路，先做可管理、可见、可启动

验收标准：
- GPT-SoVITS 能在模型控制台内完成状态查看、环境检查、手动导入承接、启动动作

---

### P3. 任务体验增强
目标：让模型部署控制台更像真正可长期使用的安装器。

子项：
1. 任务链聚合
2. 同源任务分组显示
3. 更明确的失败原因分类
4. 可选的取消 / 重试 / 只看失败任务增强

验收标准：
- 用户能看懂最近部署历史
- 能快速定位失败任务并重试

---

### P4. 下载器进一步产品化
目标：从“半自动安装”继续向更强的一键体验推进。

子项：
1. 真实下载 URL 收敛
2. 更完整的 checksum 数据
3. 断点续传增强
4. 文件级下载进度与日志细化

验收标准：
- 至少对一部分关键资源形成更稳定的自动下载与校验能力

## 当前建议执行顺序

1. **CosyVoice 半自动安装体验收尾**
2. **GPT-SoVITS 第二梯队接入**
3. **任务体验增强**
4. **下载器进一步产品化**

## 说明

当前阶段的策略是：
- **官方推荐来源优先**
- **手动导入模型目录作为兜底**
- **先 CosyVoice，后 GPT-SoVITS**

因此后续所有动作都以这条策略为主线，不再回到“纯脚本化部署”思路。
